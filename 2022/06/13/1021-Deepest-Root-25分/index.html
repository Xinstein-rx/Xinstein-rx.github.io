1021 Deepest Root 25分

2022-06-13

A graph which is connected and acyclic can be considered a tree. The height of the tree depends on the selected root. Now you are supposed to find the root that results in a highest tree. Such a root is called the deepest root.

Input Specification:

Each input file contains one test case. For each case, the first line contains a positive integer N (≤104) which is the number of nodes, and hence the nodes are numbered from 1 to N. Then N−1 lines follow, each describes an edge by given the two adjacent nodes’ numbers.

Output Specification:

For each test case, print each of the deepest roots in a line. If such a root is not unique, print them in increasing order of their numbers. In case that the given graph is not a tree, print Error: K components where K is the number of connected components in the graph.

解题思路

这个题第一遍没读懂题目，就先放了放，后来第二遍读就懂了，但是一点头绪也没有，甚至一开始看完题解也是不明白为什么要那么做，后来继续想了想，于是就想通了。自己反思没有头绪的原因是因为自己有了定式思维，认为数一定是竖着的，可是这个题就得让你横着看，先第一遍dfs找到最深的节点集合，然后从这个节点集合里面任意一个出发，以他为根节点再dfs找到最深的节点集合，这两个集合的交集，就是要求的令树最深的节点的集合。原理就是从一个最深的节点开始到另一个最深的节点，无论以这两个节点哪一个为根节点，都是令树是最高的~那如何找到这个最深的节点呢，就是在dfs的时候不断判断当前height > maxheight,如果大于就说明有更深的点，把存储最深节点的集合清空，重新把这个点push_back()进去，如果想等，就说明该深度还有别的节点，就把这个节点push_back()进去，一遍dfs之后，就找到到了最深节点，如果是只要个数的话，就记一个sum,让他不断++就好了。这个题和上面几个题解法都差不多的。
