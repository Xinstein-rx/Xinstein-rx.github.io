【动态规划】背包问题总结

2022-06-06

1 01背包模板

1.1 问题场景

有N件物品和⼀个最多能被重量为W 的背包。第i件物品的重量是weight[i]，得到的价值是value[i] 。每件物品只能⽤⼀次，求解将哪些物品装⼊背包⾥物品价值总和最⼤。

1.2 二维dp数组解法

  1. dp数组含义

dp[i][j]表示，有i件物品放到背包容量为j的背包里，最大的价值是多少

  1. dp数组如何递推

现在第i件物品，摆在一个容量为j的包面前，这个包里可能已经装有一些物品了，那第i件物品面临两种情况：

  • 一是这个包有容量可以把这个物品装进去，那到底可不可以装进去

    • 如容量j-weight[i]下，对应的最大价值即dp[i - 1][j - weight[i]]，加上当前物品的价值，大于不加入这个物品的价值即dp[i - 1][j],那么就选择加入这个物品，j-weight[i]相当于把价值不高的物品拿出来，放当前这个物品，dp数组的递推公式为dp[i][j] = dp[i - 1][j - weight[i]] + value[i];
    • 反之，如果放进去这个物品，还没有不放这个物品价值获得的多，那就选择不放入这个物品，dp数组更新为dp[i][j] = dp[i - 1][j]，即跳过这个物品

  • 二是这个包没有足够的容量把这个物品装进去那dp[i][j] = dp[i - 1][j]

  • 总结一下，dp数组的递推公式为dp[i][j] = max(dp[i - 1][j] + dp[i - 1][j - weight[i]] + value[i])

  1. dp数组的初始化
    容量为0的背包对应价值肯定为0, 即for(int i = 0; i < value.size(); i++) dp[i][0] = 0

不放入物品的背包价值肯定也为0，即for(int j = 0; j < bag_size; j++) dp[0][j] = 0

  1. dp数组的遍历顺序

有两层循环，一层遍历物品，一层遍历背包容量，遍历物品的那一层是从小到大，遍历背包的那一层也是从小打到，即从dp table的左上方更新当前dp值，可以写成

问题是，循环可不可以换过来，先遍历背包容量，再遍历物品

其实是可以的！

两层循环之间，逻辑并不会互相影响，因为dp table里每个位置，只会更新一次，并不存在覆盖问题，所以遍历谁在前谁在后，并不会互相影响。

1.2 一维数组解法

  1. dp数组含义

dp[j]表示，容量为j的背包，能装入的物品最大价值为dp[j]

  1. dp数组如何递推

与二维分析相似，每件物品都有两种状态，放入背包或者不放入背包，放不放进去，取决于两种状态

  • 一是这个背包有对应容量可以放进去，那到底放不放进去呢，要看这个物品放进去之前的最大价值即dp[j - weight[i]]，加上这个物品的价值即dp[j - weight[i]] + value[i]，是不是大于不加入这个物品的价值即dp[j]

    • 如果大于，则dp数组的递推公式为dp[j] = dp[j - weight[i]] + value[i]
    • 反之，如果放进去这个物品还没有不放进去价值大，那dp[j]选择，保留自己，不更新
    • 在这个一维的数组中，dp[j]是不断地被覆盖的，一开始遍历第一个物品的时候，dp[j]是不算被更新的，只有后面物品多了之后，才会有一些选择在里面。

  • 二是这个背包没有容量可以放进去了，那么dp[j]选择保留当前的值，即dp[j] = dp[j]

  • 总结一下,一维数组的dp递推公式为dp[j] = max(dp[j], dp[j - weight[i]] + value[i])

  1. dp数组初始化

因为只有背包容量这一个维度，所以每个容量下，还没有装入物品时，dp值设置为0即可

  1. dp数组如何遍历（很重要）

关于一维数组的遍历顺序，与二维数组有很大的不同，因为二维数组不涉及覆盖问题，一维数组因为少了一维，所以会被不断地覆盖，因此需要对遍历顺序改进

关于物品的遍历，还是从小到大，没有什么特殊的

关于背包容量的遍历，需要倒序进行遍历，即for(int j = bag_size; j >= weight[i]; j--)

倒序的原因是，不让同一个物品放进背包两次，因为如果正序遍历，随着背包容量的增大，某一件物品有容量被放进去两个，举个例子，背包容量为10，有个大小为2的物品，dp[2]的时候，已经把这个物品放进去一次了，正序遍历，到了容量为4的时候，还是会把这个物品再放进去一次，但是这是个01背包，每个物品只有一件

但是如果倒序遍历，背包的容量是在减小的，大容量的时候，我肯定能放进去大小为2的物品，而且只会放进去一次，因为我当前的dp值是之前的背包容量更新来的，之前还没放进去这个大小为2的物品呢，所以放就只放进去一次

那如何证明这个方法一定是正确的呢？

因为根本的逻辑没变，还是当前背包容量，能不能装进去这个物品，能不能装这件事，看的是当前容量之前的哪些情况，即dp[j - weight[i]]，而不会看后面那些容量，所以只是遍历顺序上有些不同，根本逻辑还是不变的，所以这个方法肯定是正确的

所以遍历顺序可以写成

那这两个遍历顺序可不可以换过来，先遍历背包容量，再遍历物品呢，这次就不可以了！

因为dp[j]是要被不断覆盖掉的，如果先遍历背包容量，他的遍历逻辑就是当前容量下，我能装入的最大的一个物品的价值，是只能装进去一个，因为遍历第一遍的时候，只是最大的dp[bag_size]的到了更新，前面的容量都没有被更新到值

2 01背包变式应用

2.1 物品的分割

01背包可用于物品分割这一场景，无论是什么东西的分割，只要是从一堆东西里面，选出一些满足要求（最大价值，最小XX，能不能凑成）的东西，每件东西只能选择一次，即物品选择性问题，都可以用01背包的套路来求

2.1.1 分割等子集

题目链接：https://leetcode.cn/problems/partition-equal-subset-sum/

2.1.2 最后一块石头的重量

题目链接：https://leetcode.cn/problems/last-stone-weight-ii/

这类题目关键就是定义好物品的价值与重量，以及背包的容量是多少，很多情况下，物品的容量和他的价值是相同的。

2.2 凑成某个值的种类数

即凑成某个值，就是要装满背包，而不用管是不是最少的物品，最多的物品，那就是要求，装满背包的方案数字。

这个不是简单的模板运用了，里面主要的思想就是，如果有个物品可以装入背包，那么把这件物品装进背包所得到的方案数，与不装入这件物品的方案数，是相同的，这里的dp[j]表示，装满容量为j的背包方案数是dp[j]。即dp[j] = dp[j - weight[i]]，因为装入这件物品，并没有改变方案数。

不同的物品加进来，都算是不同的方法，每个容量的方案数，是减去不同的物品，即放入不同的物品的累积

举个例子，一开始第一个物品的时候，假设重量是3，只有能正好能装下这个物品的背包容量dp值为1，即dp[3] = 1，其余位置都为0

第二个物品进来，假设重量是1，则dp[4]可以由dp[3]更新来，dp[4] = dp[3], 当然了，这里dp[1] = 1

第三个物品进来，假设重量是2，那这时候dp[3]就有意思了，它可以装进去第一个重量为3的物品，也可以装入重量为1， 2的两个物品，有两种方案了，所以dp[3] = dp[3] + dp[3 - 2]

所以总结一下，每个容量更新的值，都是当前能凑成的数，加上新进来的物品，装进这个物品之前，有多少种方法，即 dp[j] += dp[j - weight[i]]

2.3 求装入背包种物品的数量

2.3.1 1和0

题目链接：https://leetcode.cn/problems/ones-and-zeroes/

这道题与普通的01背包还不太一样，这个背包有了两个容量，两类物品，需要同时满足这两个容量条件

这类题目不是要求最大价值了，而是要求加入背包的最多的物品的数量，一开始的时候，背包容量都为0，每来一个物品，要不要装进去，就是看装进去之前，对应容量+1，+1是因为把这个物品装进去了，是不是大于不装这个物品当前的数量，即dp[j] = max(dp[j], dp[j - weight[i]] + 1)

这里因为不牵扯价值，只是无脑往背包里装越多的物品就好，所以每次更新都是+1

3 完全背包模板

3.1 问题场景

有N件物品和⼀个最多能被重量为W 的背包。第i件物品的重量是weight[i]，得到的价值是value[i] 。每件物品可以用无数次，求解将哪些物品装⼊背包⾥物品价值总和最⼤。

完全背包和01背包唯一不同的地方，就是物品可以使用多次。

  1. dp含义，与01背包相同，dp[j]都是代表容量为j的背包可以装入物品的最大价值

  2. dp的更新，也与01背包相同，dp[j] = max(dp[j], dp[j - weight[i]] + value[i])，因为无论有多少个物品，都要看要不要放下这个物品

  3. dp的遍历顺序，这是唯一与01背包不同的地方，之前01背包的时候，背包容量是从大到小遍历，是因为避免重复把物品放进来，那现在有重复物品了，所以背包容量可以从小到大，而且也必须从小到大进行遍历即for(int j = weight[i]; j <= bag_size; j++)

  4. dp的初始化也与01背包相同，均为0

总结一下，完全背包的模板为

那在上面这个遍历顺序下，是先遍历物品再遍历背包容量，那这两个遍历顺序可不可以换一下呢，在这里是可以换的！！01背包里面不可以换

因为遍历顺序是从小到大的，后面dp值的更新依赖于前面，而前面已经是被完全更新，不会被覆盖掉的了，也不会影响后面dp值的计算，所以遍历顺序是可以换的。

3.2 求组合数

3.2.1 零钱兑换Ⅱ（组合数整体的个数）

题目链接：https://leetcode.cn/problems/coin-change-2/

这个是求装满背包的方法的数量，核心思想就是dp[j] += dp[j - weight[i]]

组合数要求，先遍历物品，再遍历背包，因为组合数不要求数据有顺序，所以只是选出来就可以了{1，2}，{2， 1}算作一种情况，所以这样的遍历顺序，保证了只会计算到{1， 2}，不会计算到{2，1}，就不会重复计算，因为物品先遍历的特点，就保证了2这个物品一定会出现在1之后，不会出现在之前。

3.3 求排列数

3.3.1 组合总和

题目链接：https://leetcode.cn/problems/combination-sum-iv/

排列数要求，必须先遍历背包，再遍历物品，因为排序要求有顺序，如果反过来，先遍历物品，那么物品组合{1， 2}会出现，{2， 1}这种情况并不会出现，要保证{2， 1}也出现的话，就必须是先遍历背包，再遍历物品

这个是求装在背包里物品的个数，并且背包是装满的，递推公式就不是加value了，而是加1就可以了，代表放进去了一个物品

3.4.2 完全平⽅数

题目链接：https://leetcode.cn/problems/perfect-squares/

3.4.3 单词拆分

题目链接： https://leetcode.cn/problems/word-break/

上面题目的特点是，求组合数和排列数都可以，只是求一个数量，或者是不是存在，所以，遍历顺序是可以互换的

4 多重背包模板

4.1 问题场景

有N件物品和⼀个最多能被重量为W 的背包。第i件物品的重量是weight[i]，得到的价值是value[i] 。每件物品可以用Mi次，求解将哪些物品装⼊背包⾥物品价值总和最⼤。

多重背包和01背包很像，每件物品最多有Mi件可⽤，把Mi件摊开，其实就是⼀个01背包问题了。

处理的时候完全可以先整理出一个vector，里面是所有的物品（有重复），然后跑01背包的模板
