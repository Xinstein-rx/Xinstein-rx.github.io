1094 The Largest Generation (25分)

2020-05-26

A family hierarchy is usually presented by a pedigree tree where all the nodes on the same level belong to the same generation. Your task is to find the generation with the largest population.

Input Specification:

Each input file contains one test case. Each case starts with two positive integers N (<100) which is the total number of family members in the tree (and hence assume that all the members are numbered from 01 to N), and M (<N) which is the number of family members who have children. Then M lines follow, each contains the information of a family member in the following format:

ID K ID[1] ID[2] ... ID[K]

where ID is a two-digit number representing a family member, K (>0) is the number of his/her children, followed by a sequence of two-digit ID‘s of his/her children. For the sake of simplicity, let us fix the root ID to be 01. All the numbers in a line are separated by a space.

Output Specification:

For each test case, print in one line the largest population number and the level of the corresponding generation. It is assumed that such a generation is unique, and the root level is defined to be 1.

解题思路

这个题和前面的一个三十分的题很像，然后自己对于树的遍历可能是比较薄弱，希望通过这个题可以好好巩固一下，这个题有三种方法可以做。第一种是dfs也是代码比较简单的一个方法，就是不断往下深搜，定义一个book数组，记录每层节点的个数，dfs传的参数是id，和level，id就是在深搜的时候循环里的孩子节点的id，level就是上一层level + 1，也就是父节点的level + 1，然后深搜完之后，book数组也就建好了，再遍历book数组，找到最大的。第二种是bfs广搜，我看广搜都是定义了一个结构体，从队列中pop出来一个节点，然后遍历这个节点的孩子，把他们孩子节点的level都变成父节点level + 1，然后从1 到 n遍历，把对应level的book数组算出来，再遍历book数组，找到最大的。第三种方法是直接用vector存对应节点的孩子，然后再定义两个向量va，vb，va存的是父节点的id，vb存的是孩子节点的id，三层循环，第一层是level，第二层是遍历va，也就是对应那一层的父节点，第三层是遍历每个父节点的孩子节点，把他们都push_back()到vb里，最后va = vb，作为下一层的父节点，vb.clear(),每次比较vb的大小就可以了。补充：刚才试了试不用结构体的bfs，就是定义一个level[]数组，然后也开心的过了，不过开始因为输入的不对没过，还让我的贤贤帮忙找bug呜呜呜，低级错误。
